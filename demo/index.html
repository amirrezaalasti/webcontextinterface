Human View vs. Agent View

The same web — two radically different representations.

Feature 🧑 Human-Friendly (Standard) 🤖 Agent-Friendly (AgentDOM)
Structure Deeply nested, CSS-dependent divs Flat, schema-typed node list
Navigation Visual cues — icons, colors, hover Descriptive data-agent-desc
State Implicit — URL changes, CSS classes Explicit ActionResult payloads
Noise High — ads, navbars, hero images Low — only functional nodes
Errors Red border, toast notification Typed error.code + error.hint
Context cost50–500 KB raw HTML 1–10 KB distilled JSON/MD
Side effectsRe-parse full DOM sideEffects[] in ActionResult
Auth Implicit redirect flow Auth-Required in agents.txt
Scoping None — full DOM exposed Landmark scopes bound reasoning

Site Context Layer

Like robots.txt — but for agents. Three files at the web root give LLMs site-wide grounding before they touch a single page.

📄 /agents.txt

Directive file — allow/deny scopes, rate limits, auth requirements, human-confirmation guards. Machine-parsed by the agent runtime.

View file ↗
📋 /agents.json

Structured site manifest — capabilities, authentication flows, multi-step task flows, scope descriptors. For tool-calling agents.

View file ↗
📖 /agent.md

Rich Markdown narrative injected into the LLM's system prompt. Explains purpose, allowed actions, task flows, and error recovery.

data-agent-* Attribute Reference

12 attributes. Zero breaking changes. HTML5-compatible.

AttributeTypeDescription
data-agent-id string Stable unique identifier for this node
data-agent-role enum action · form · display · nav · status · landmark
data-agent-desc string LLM-optimised description of what this node does
data-agent-action enum click · fill · select · check · upload · submit · navigate
data-agent-state JSON Current observable state snapshot
data-agent-preconditionstring Natural language guard condition
data-agent-required boolean Whether this field must be satisfied before form submission
data-agent-options JSON arrayEnumerated choices for select/radio/checkbox groups
data-agent-emit string Custom DOM event fired after interaction
data-agent-scope string Parent landmark scope ID
data-agent-hidden boolean Flag decorative nodes for Distiller pruning
data-agent-priority 1–5 Importance ranking within scope (1 = primary CTA)
