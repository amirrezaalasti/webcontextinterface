\noindent Model slugs, temperature ($0$), \texttt{max\_tokens} ($800$ multi-step / $1000$ single-shot), reasoning effort (\texttt{low}), and exact system prompts are documented in \texttt{docs/benchmark-eval-config.md} and exported to \texttt{demo/public/eval-config.json} from \texttt{evals/lib/eval-config.ts}.
\par\noindent For the assessment, all approaches are scored against a unified pass rule requiring three conditions to be met simultaneously: (1) the \texttt{final\_action} must correctly resolve to the ground-truth element in a headless Chromium instance, (2) the selected element must not be a predefined semantic decoy or competitor trap, and (3) the generated JSON plan must achieve a semantic flow coverage score of at least 0.6 when compared to the expected task trajectory. Non-WCI outputs were validated using Playwright-based execution in headless Chromium, while WCI outputs required exact \texttt{wciNodeId} matching against annotated semantic nodes.
\par\noindent Semantic flow coverage is:
\[
\begin{aligned}
C_{\mathrm{flow}}&=\max\!\left(C_{\mathrm{type}},\,C_{\mathrm{step}}\right),\\
C_{\mathrm{type}}&=\frac{|B_{\mathrm{exp}}\cap B_{\mathrm{obs}}|}{|B_{\mathrm{exp}}|},\\
C_{\mathrm{step}}&=\frac{1}{|S_{\mathrm{exp}}|}
\sum_{s\in S_{\mathrm{exp}}}\mathbf{1}\{\mathrm{match}(s,\hat{P})\}.
\end{aligned}
\]
Here, \(B_{\mathrm{exp}}\) are expected flow-type buckets, \(B_{\mathrm{obs}}\) are buckets inferred from plan \(\hat{P}\), and \(S_{\mathrm{exp}}\) are expected steps. A step matches if type and/or lexical cues align. Inference uses explicit action labels, step/target text, and constrained \texttt{final\_action}-based credit; under WCI, correct grounding can impute expected observe/verify buckets, while non-WCI baselines require explicit evidence.
